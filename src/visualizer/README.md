# Visualizer Module: Terminal UI and State Rendering

The `visualizer` directory provides real-time, terminal-based feedback during the engine's autoregressive generation loop. In a standard pipeline, the LLM operates as a black box until the final string is returned. This module cracks that box open, acting as a pedagogical window into the low-level mechanics of constrained decoding. It renders CFG state changes, dynamic logit mask sizes, and the final structural output via a clean ASCII tree, allowing developers to see exactly *how* the engine is forcing compliance millisecond by millisecond.

## Theoretical Concepts

### Standard Output Buffering vs. Flushing

By default, Python buffers standard output (`sys.stdout`) and only writes to the terminal when it encounters a newline character (`\n`). During high-speed token generation, waiting for newlines makes the UI feel laggy or clunky. The visualizer explicitly overrides this by forcing a buffer flush (`flush=True`), ensuring the terminal updates the instant the LLM selects a token.

### The Carriage Return (`\r`)

If a script prints a new line for every single token generated by an LLM, the terminal history will be flooded with hundreds of useless lines within seconds. The carriage return (`\r`) is a low-level control character that moves the cursor back to the absolute beginning of the *current* line without dropping down. This allows the visualizer to continuously overwrite the same line, creating a smooth, static UI dashboard at the bottom of the screen.

### ANSI Escape Codes

Terminals do not inherently understand colours or bold text. To change the appearance of a string, specific byte sequences (ANSI escape codes) must be injected directly into the text stream before the characters are printed. The visualizer coordinates with the project's internal `Formatter` utility to dynamically wrap variables in these byte codes.

## Design Decisions

### Stateless Architecture

The `Visualizer` is implemented entirely using `@staticmethod`s. It holds no internal state, memory, or model references, making it completely decoupled and safe to inject anywhere in the pipeline without risking memory leaks.

### Line Clearing (`\033[K`)

When overwriting a line via carriage return, if the new string is shorter than the old string, ghost characters will remain visible. The visualizer prepends the ANSI sequence `\033[K` (Clear to end of line) before writing the new status, ensuring a perfectly clean render every frame.

### Safe Token Rendering

LLMs frequently generate raw control characters (like `\n` or `\t`) inside JSON strings. If printed directly, these would instantly break the carriage return UI. The visualizer actively intercepts and escapes these (`replace('\n', '\\n')`) into literal, printable string representations.

### Fallback Highlighting

The visualizer is tightly integrated with the `PostProcessor`'s graceful degradation logic. If the engine injects a safe fallback object due to a parsing failure, the visualizer explicitly flags the injected `error` key in red, preventing silent failures during verbose debugging runs.

## Challenges Solved

### Terminal Flooding

Completely eliminated by the carriage return and line-clearing architecture, keeping the developer's console clean even when generating thousands of tokens.

### Black-Box Debugging

When a constrained decoding engine fails, it is incredibly difficult to know *why* without seeing the math. By printing the current CFG `State` and the exact number of `allowed chars` remaining in the mask, the visualizer bridges the gap between the logit probabilities and the JSON logic, making it instantly obvious if a schema rule is misaligned.

### Readability of Output

Raw JSON strings are dense and difficult to read quickly. The `print_json_render` method parses the output and draws a hierarchical ASCII tree, drastically improving visual parsing speed during peer evaluations or batch testing.

## Glossary

### Carriage Return (`\r`)

A control character that commands the terminal cursor to return to the beginning of the active line.

### ANSI Escape Sequences

Standardised codes embedded in text streams to control terminal formatting, such as colour, bolding, and cursor positioning.

### Flush

Forcing the output buffer to immediately write its contents to the terminal display, overriding default line-buffering behaviours.

### ASCII Tree

A textual representation of hierarchical data (like a JSON object) using standard line-drawing characters (e.g., `├─`, `└─`) to indicate parent-child relationships.

### State Machine (CFG)

The internal tracking logic that dictates what structural characters are legally allowed at the current moment in the generation.
